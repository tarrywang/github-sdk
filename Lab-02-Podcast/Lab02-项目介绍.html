每日播客自动生成

用 Microsoft Agent Framework（MAF） 编排三个 GitHub Copilot Agent：内置爬虫从央视体育抓取 5 条世界杯新闻，串联生成一档「双人对谈」播客脚本。

概览 · 这个项目是什么
02

一句话：爬当天新闻， 让三个 AI 接力写成一档播客。

输入

实时新闻（自动爬取）

内置爬虫从 CCTV 体育国际足球频道 抓 5 条最新美加墨世界杯新闻并提取正文。无需手动传话题。

处理

三 Agent 串联

搜索/大纲 → 撰稿 → 润色，每个 Agent 都由 GitHub Copilot 作为 LLM 驱动，MAF 负责编排传递。

产出

双人对谈脚本

Host（黄健翔式激情）× Guest（贺炜式文艺）的对话脚本，存为 podcast/2p_podcast_<uuid>.txt。

💡 与 Lab 01 的不同　Lab 01 用 gh-aw（Markdown 工作流跑在 Actions 里）；Lab 02 用 MAF —— 一个 Python 程序，把多个 Agent 编排成一张执行图，本地一条命令即可跑通，也可挂到 Actions 上每日定时。

概览02
核心概念 · MAF 是什么
03

三个 关键抽象， 撑起整条工作流。

抽象 01

Executor

工作流里的一个执行节点。本项目每个 Executor 内部封装一个 GitHubCopilotAgent，用 @handler 标注处理方法。

抽象 02

WorkflowBuilder

用 .add_edge(a, b) 把 Executor 连成有向图，start_executor= 指定入口，.build() 产出可运行的 Workflow。

抽象 03

WorkflowContext

ctx.send_message() 把结果传给下一个节点；末端用 ctx.yield_output() 产出工作流最终结果。

GitHubCopilotAgent 是 MAF 提供的 LLM 适配器，通过 Copilot CLI 调用模型（默认 gpt-5.4）——所以无需 Azure AI Foundry 资源，有 Copilot 订阅 + gh auth login 即可。

搭建步骤 · 详解 ②
06

挂上 GitHub Actions，每天自动播报。

① Fork & 配 Token

建 Fine-grained PAT，Copilot Requests = Read and Write（注意 Lab 02 要写权限，比 Lab 01 多一个 Write）；存为 Secret COPILOT_GITHUB_TOKEN。

② 触发方式

Actions 页手动触发 Daily Podcast Generator，或等每日定时 —— cron 0 0 * * *（UTC 00:00 = 北京 08:00）。

③ 工作流做了什么

checkout → 装 Python 3.11 → pip install --pre → 跑 podcast_workflow.py → 自动 commit & push 生成的播客文件。

④ 权限说明

工作流 permissions: contents: write —— 因为它要把生成结果直接提交回仓库（不像 Lab 01 走 safe-outputs PR）。

设计 · 最值得学的一点：提示工程
09

三个 Agent = 三套精心设计的人设提示词。

AGENT 1 · 策划人
Search
「专业体育播客内容策划人」——按真实新闻提炼要点、生成大纲，强约束「不要杜撰新闻里没有的事实」。
AGENT 2 · 撰稿人
Content
「双人对谈撰稿人」——Host=黄健翔式激情、Guest=贺炜式文艺；规定每人每次 2–3 句、你来我往。
AGENT 3 · 编辑
Script
「脚本编辑」——最终润色，保持双人风格与节奏，直接输出成稿不加说明。

💡 为什么这是绝佳的提示工程范例　三个 system instructions 演示了「角色一致性」——给 AI 清晰的身份、风格、硬约束（句长/不杜撰/输出格式），产出质量天差地别。对照练习：差提示「帮我写个爬虫」 vs 好提示「为 crawl_world_cup_news 加重试：失败 URL 最多重试 2 次、指数退避、仍失败则跳过并记日志」。

GitHub Copilot · 在本项目中的角色
10

Copilot 既是 项目的引擎，也是你的副驾。

i.
Agent 的 LLM
三个 GitHubCopilotAgent 通过 Copilot CLI 调模型（gpt-5.4）—— 它就是大纲、脚本、润色背后的「笔」。
→ 项目运行时的大脑
ii.
IDE 编程副驾
VS Code 里 Chat / Edit / Agent 模式、Ghost Text，帮你读懂编排、改爬虫、写测试、补文档。
→ 搭建时的助手
iii.
终端命令助手
ghcs 建议命令、@terminal 排错 —— 跨平台 venv / 安装命令记不住就问它。
→ 命令行的拐杖

🛡 安全底线：Copilot 会犯错，也可能生成不安全或不合规的代码。务必① 审阅每个建议；② 用测试验证；③ 不盲目接受；④ 关注第三方许可证。把它当副驾，方向盘始终在你手里。

Copilot Tips · 搭建中顺手练到的功能
11

本 Lab 主打：提示工程 · Edit · Agent 模式。

提示工程 · 好 vs 差本 Lab 核心

对比三个 Agent 的 system instructions 体会「角色一致性」；再对照 「帮我写个爬虫」 vs 「为 crawl_world_cup_news 加重试、指数退避、失败跳过并记日志」 —— 后者产出明显更可用。

@workspace读懂编排

问「@workspace podcast_workflow.py 如何用 WorkflowBuilder 把三个 Executor 串起来？」→ 快速建立全局认识。

Edit 模式多文件 diff

选中 crawl_world_cup_news 的 asyncio.gather(...)，说「加 return_exceptions=True 与单条超时，个别失败不影响整体」→ 给 diff 逐条确认。

Agent 模式自主多步

CCTV 页面结构变了？切 Agent 模式（Win Ctrl+Shift+I / mac Shift+Cmd+I）说「适配新 HTML，更新 ListParser 与 ArticleParser 并自测」→ 跨多处自主修改，风险命令先问你。

/explain · /tests读正则 + 生成用例

选中 ARTICLE_URL_PATTERN 用 /explain 看懂这条 CCTV URL 正则，再 /tests 生成用例（人工核对断言）。

/doc · @terminal · ghcs补文档 + 排错

选中 crawl_world_cup_news 用 /doc 生成中文 docstring；跨平台命令记不住 ghcs "activate a python venv on Windows"；装包报错贴进 Chat 问 @terminal。
